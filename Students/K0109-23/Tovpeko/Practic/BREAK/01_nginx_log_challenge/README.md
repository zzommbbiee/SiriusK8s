
# Как я генерировал тестовые логи nginx

Я хотел потренироваться в анализе логов — например, найти топ-10 IP по количеству запросов. Для этого понадобились большие файлы `access.log` и `access_json.log`. Я взял готовый скрипт и запускал его по шагам.

----

## Что делал

1. Создал папку:  
   ```bash
   mkdir -p ~/nginx-challenge && cd ~/nginx-challenge
   ```

2. Сделал пустой `_common.sh`, потом понял — нужно его заполнить, иначе скрипт не запустится.

3. Написал простой `_common.sh` с функциями `banner`, `require_root`, `require_linux` — чтобы скрипт не упал на старте.

4. Запустил `sudo bash script.sh` — сначала выдало ошибку `banner: команда не найдена`, потому что `_common.sh` был пустой.

5. Потом при копировании `cat << 'EOF'` сломал отступы — получилась ошибка про `EOF`, и опять не запустилось.

6. В третий раз вставил `_common.sh` правильно — уже прошло дальше, но Python упал с `NameError: name 'i' is not defined`. в коде была ошибка с переменной `i`.

7. Исправил код питона. После этого скрипт отработал — создал два лога (37 и 200 тыс строк), всё ок.

8. Проверил:
   - `ls -lh /var/log/nginx/` — файлы есть,
   - `systemctl reload nginx` — логи снова пишутся нормально.

## Примеры команд, которые я использовал

### Работа с файлами
```bash
# Посмотреть размер логов
ls -lh /var/log/nginx/access*.log

# Удалить тестовые логи (если нужно начать заново)
sudo rm -f /var/log/nginx/access.log /var/log/nginx/access_json.log
```

### Анализ логов (то, ради чего всё делалось)
```bash
# Топ-10 IP из обычного access.log
awk '{print $1}' /var/log/nginx/access.log | sort | uniq -c | sort -nr | head -10

# Сколько всего строк в логах (проверка, что генерация прошла)
wc -l /var/log/nginx/access.log
wc -l /var/log/nginx/access_json.log
```

### Восстановление после тестов
```bash
# Вернуть права на каталог логов (на всякий chown root:adm /var/log/nginx
sudo chmod 755 /var/log/nginx

# Перезагрузить nginx, чтобы он начал писать в чистые логи
sudo systemctl reload nginx

```
у меня ещё оказывется не был изначально установлен nginx на виртуалке, на одном из скринов видно, потом докачал и заного запустил скрипт.

 Ошибки, с которыми столкнулся

- **`banner: команда не найдена`** — просто `_common.sh` был пустой.
- **`EOF` не найден** — неправильно скопировал многострочный `cat << 'EOF'`.
- **`NameError: name 'i' is not defined`** — баг в оригинальном Python-коде (переменная `i` не объявлена).

Всё исправил вручную, ничего не сломал — только потренировался 😊
