# Как я восстанавливал сеть после скрипта 2.sh

После запуска `2.sh` сеть сломалась — нет интернета, не открываются сайты, SSH не отвалился (не стал разбираться почему, но интересная тема, можно будет поинтересоваться). Я восстанавливал всё из бэкапов, которые лежали у другого юзера.

----
ФОТО С РАБОТОЙ В ПАПКЕ /images
----

## Что делал

1. Зашёл в систему.

2. Нашёл бэкапы, у другого пользователя.

3. Скопировал их обратно:
   - `scp urahoy-2@debian-2:/etc/resolv.conf /tmp/resolv.ok`
   - `scp user@other-host:/etc/nsswitch.conf /tmp/nsswitch.ok`

4. Восстановил маршруты:
   ```bash
   ip route add default via 192.168.122.1 dev enp1s0
   ```

5. Проверил, что всё заработало:
   ```bash
    ping -c 2 8.8.8.8
   ```

## Что сломалось

- DNS указывал на `203.0.113.254` (фиктивный).
- В `/etc/nsswitch.conf` был только `files`, без `dns`.
- Маршрут по умолчанию был удалён.

## Что починил

- Восстановил правильный `/etc/resolv.conf` из бэкапа.
- Восстановил switch.conf.
- Прописал статику, так как это ВМ и знаю её шлюз.

Всё, сеть снова работает как раньше.
