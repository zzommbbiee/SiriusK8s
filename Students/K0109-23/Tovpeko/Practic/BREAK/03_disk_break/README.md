
# Как я освобождал место на диске

После запуска скрипта место на диске стало заканчиваться — он создавал два файла, которые занимали место, и один из них был «удалён», но всё ещё держался процессом.

----
ФОТО С РАБОТОЙ В ПАПКЕ /images
----

## Что делал

1. Проверил, сколько места занято:
   ```bash
   df -h /
   ```
   Видно, что почти весь диск занят.

2. Нашёл большой файл:
   ```bash
   du -sh /var/lib/linux_break_lab/*
   ```
   Видно, что `disk_fill.bin` занимает много места.

3. Удалил его:
   ```bash
   sudo rm -f /var/lib/linux_break_lab/disk_fill.bin
   ```

4. Узнал, что есть процесс, который держит удалённый файл:
   ```bash
   cat /var/lib/linux_break_lab/orphan_holder.pid
   ```

6. Посмотрел, какие файлы он держит:
   ```bash
   sudo lsof -p $(cat /var/lib/linux_break_lab/orphan_holder.pid)
   ```
   Видно, что есть файл с пометкой `(deleted)` — это и есть «невидимое» место.

7. Убил процесс:
   ```bash
   sudo kill $(cat /var/lib/linux_break_lab/orphan_holder.pid)
   ```

8. Проверил, что место освободилось:
   ```bash
   df -h /
   ```

## Что было сломано

- Большой файл `disk_fill.bin` занимал место на диске.
- Процесс держал удалённый файл, и место не освобождалось после `rm`.

## Что починил

- Удалил видимый файл.
- Убил процесс, державший удалённый файл — место освободилось.

Всё, диск снова свободен.
