# 2. Починить сеть + применение find / xargs / awk / grep

Ломаются default route и DNS (подменяется `resolv.conf`, отключается DNS в `nsswitch.conf`), затем вручную восстанавливается маршрут через `ip route add` и резолвинг через `echo nameserver 8.8.8.8 > /etc/resolv.conf`.

При диагностике и поиске проблем активно применяются **find** (поиск конфигурационных файлов по имени и времени изменения), **xargs** (передача найденных файлов в команды обработки), **awk** (извлечение и изменение полей в `/etc/nsswitch.conf`), **grep** (фильтрация строк и контекстный поиск ошибок в логах). Это показывает, как стандартные утилиты Unix помогают быстро анализировать и чинить сетевые конфигурации.
