# 4. Незапускаемый бинарник + анализ TCP handshake (tcpdump + tmux)

**Незапускаемые бинарники:** создаются несколько исполняемых файлов с разными дефектами (без права на выполнение, неверный shebang, усечённый ELF, битый интерпретатор). С помощью `file`, `chmod`, `ldd`, `strace` диагностируются и исправляются проблемы.

**Анализ трафика:** в одной панели **tmux** запускается `tcpdump` с фильтром `host google.com and port 443`, в соседней — `curl -v https://google.com`. В дампе находится TCP handshake: пакеты с флагами `[S]` (SYN), `[S.]` (SYN-ACK), `[.]` (ACK). Также обнаруживаются TLS Client Hello и Server Hello (трафик зашифрован, но установка соединения наглядно показана). Запись asciinema подтверждает навыки сетевой диагностики и работы с бинарными файлами.
