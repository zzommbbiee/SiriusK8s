Отчет по лабораторной работе №3: Docker: сети, volumes, docker-compose
1. Чему научился (Результаты работы)
В ходе выполнения лабораторной работы были освоены принципы организации многоконтейнерных приложений с использованием Docker Compose.

Изучены типы сетей Docker: bridge (изолированная сеть для контейнеров на одной ноде), host (контейнер использует сеть хоста), overlay (для связи контейнеров на разных хостах в кластере Swarm). Создана пользовательская bridge-сеть и проверено, что контейнеры в ней видят друг друга по имени через встроенный DNS.

Получен практический опыт работы с persistent volumes: создан volume для PostgreSQL, данные успешно сохранились после удаления и пересоздания контейнера.

Написан docker-compose.yml для стека из трёх сервисов: база данных PostgreSQL, бэкенд на Flask и фронтенд на nginx. Настроены healthcheck для каждого сервиса, что гарантирует правильный порядок запуска.

Стек успешно поднят, проверена работа цепочки: внешний запрос к nginx → бэкенд → база данных. Данные из БД возвращаются в формате JSON.

Выполнено масштабирование бэкенда до трёх экземпляров с помощью docker compose up --scale backend=3. Все три контейнера работают параллельно.

2. Возникшие проблемы и способы их решения
В ходе выполнения работы проблем не возникло. Все шаги были выполнены успешно с первого раза благодаря чётким инструкциям и корректной настройке зависимостей и healthcheck-ов.
