Отчет по модулю №1: Процессы, systemd и анализ логов

В первой части работы я изучала механизмы управления процессами в Linux и принципы работы systemd. Z использовала файловый менеджер Midnight Commander для подготовки структуры директорий в каталоге /opt и текстовый редактор Vim для написания скриптов и конфигурационных файлов. Основной задачей было создание отказоустойчивого сервиса, способного автоматически восстанавливаться после критических сбоев. Я написала Bash-скрипт и подготовила соответствующий unit-файл в директории /etc/systemd/system, установив параметр Restart=always. В ходе тестирования я намеренно завершила основной процесс сигналом SIGKILL, после чего через статус сервиса и логи journalctl убедилась, что systemd мгновенно зафиксировал падение и успешно перезапустил службу с новым PID.

Вторая часть работы была посвящена практическому анализу больших объемов данных с использованием стандартных утилит обработки текста. Используя генератор на базе Python, я создала логи веб-сервера Nginx объемом в миллион строк, чтоб cымитировать реальную нагрузку на систему. Для решения задачи по поиску топ-10 самых активных IP-адресов я выстроила конвейер из утилит awk, sort и uniq. Сначала я извлекла первый столбец с адресами из классического access.log, выполнила их сортировку и подсчет уникальных вхождений, после чего отфильтровала результат по количеству запросов. Такие же действия я применила для лога в формате JSON, адаптировав команду awk для корректного выделения поля remote_addr по специфичному разделителю. Завершающим этапом стала очистка системы от временных данных, что подтвердило успешное освоение инструментов фильтрации и мониторинга файловой системы.
