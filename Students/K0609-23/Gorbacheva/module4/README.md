Отчет по модулю №4: Сеть: curl, ss, tcpdump, nc

В первой части модуля я работала с утилитами сетевой диагностики для анализа трафика на транспортном и прикладном уровнях. Используя tmux, я параллельно запустила tcpdump (с флагом -A для отображения содержимого пакетов) и инициировала HTTP-запрос утилитой curl по 80 порту. Это позволило перехватить открытый трафик и наглядно отследить процесс установки TCP-соединения (Three-way handshake: пакеты с флагами SYN, SYN-ACK, ACK), а сразу после рукопожатия — зафиксировать передачу HTTP-заголовков, включая ответ сервера со статусом 301 Moved Permanently.
https://asciinema.org/a/2OD9wVHS4JmTj6qm

Вторая часть была посвящена траблшутингу исполняемых файлов, возвращающих неочевидные системные ошибки при запуске. Я проанализировала директорию с намеренно поврежденными скриптами и бинарниками. Отсутствующий бит исполнения был восстановлен командой chmod, а битый shebang в текстовом скрипте удалось обойти явным вызовом через интерпретатор bash. Использование утилиты file помогло выявить физически усеченный заголовок ELF-файла, из-за которого ядро отказывалось его запускать. Самым показательным кейсом стал динамический бинарник, выдававший ложную ошибку «No such file or directory»: с помощью утилиты ldd и трассировщика системных вызовов strace я доказала, что сбой происходит из-за попытки загрузить несуществующий интерпретатор динамических библиотек.
