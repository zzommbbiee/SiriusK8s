Отчет по модулю №2: Диски, файловые системы и сетевая маршрутизация

В первой части работы я изучала инструменты управления дисковым пространством и особенности работы файловых систем на уровне индексных дескрипторов (inode). Для имитации реальной задачи я создала массив тестовых данных утилитой fallocate, искусственно состарив часть файлов с помощью изменения временных меток командой touch. Главной целью было автоматизировать поиск и удаление логов старше семи дней и размером более 100 МБ. Я решила эту задачу с помощью конвейера из утилит find и xargs, где find по заданным фильтрам сформировал список целевых файлов, а xargs передал их на безопасное удаление. Замеры утилитой du -sh до и после выполнения команды подтвердили успешное освобождение дискового пространства. Дополнительно я использовала утилиту stat для детального разбора структуры метаданных файла, что продемонстрировало понимание внутреннего устройства файловой системы.

Во второй части модуля я проводила диагностику и восстановление сетевой связности виртуальной машины после программного сбоя. Первичная проверка доступности ресурсов утилитой curl завершилась ошибкой, что потребовало пошагового анализа сетевого стека. Проверка таблицы маршрутов командой ip route выявила полное отсутствие маршрута по умолчанию. Я восстановила базовую связность, вручную добавив корректный шлюз (default gateway) для моей подсети. Однако разрешение доменных имен по-прежнему не функционировало. Отладка показала, что файл /etc/resolv.conf содержал невалидный IP-адрес DNS-сервера, а в файле системных приоритетов /etc/nsswitch.conf была удалена директива dns для поиска хостов. Я отредактировала оба конфигурационных файла, вернув приоритет разрешения имен через внешние серверы. Сделала итоговый тестовый запрос для подтверждения работы.
