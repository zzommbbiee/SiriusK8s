Отчет по модулю №5: Мониторинг: top, atop, htop, ps, strace

В первой части, используя tmux, я создала среду для комплексного анализа. Пока утилита stress имитировала высокую нагрузку на CPU и память, я визуализировала потребление ресурсов в htop и проводила трассировку системных вызовов через strace -p. Это позволило мне в реальном времени наблюдать за взаимодействием приложения с ядром Linux (вызовы управления памятью и потоками) при критической нагрузке.
https://asciinema.org/a/dhnVXzUpXzpP4pTQ

Во второй части я провела исследование поведения падающего сервиса под управлением systemd. Был настроен юнит script-server.service, запрограммированный на немедленное завершение с ошибкой. С помощью systemctl status я зафиксировала цикличный перезапуск службы, а через journalctl проанализировала журналы событий, выявив некорректный код возврата процесса. Данный опыт закрепил навыки оперативной локализации неисправностей в инфраструктуре и анализа логов для восстановления работоспособности ОС.
