# Практическая задача 02 — починить сеть

## Задача
Восстановить доступ сервера к интернету при наличии только известного ip-адреса. Требовалось настроить маршрутизацию, шлюз и dns.

## Решение
Для решения задачи я настроила сетевые параметры системы. Сначала добавила маршрут по умолчанию через шлюз, используя команду `ip route`. Затем сконфигурировала файл `/etc/resolv.conf`, прописав в нем адреса DNS-серверов, и проверила настройки в `nsswitch.conf`. Итоговую проверку связи выполнила с помощью запроса `curl www.avito.ru`.

## Сложности
Основная сложность заключалась в том, что без одновременной настройки и шлюза, и dns-серверов, доступ к внешним ресурсам по доменным именам оставался невозможным. Важно было проверить правильность заполнения конфигов, чтобы система могла корректно резолвить адреса.
