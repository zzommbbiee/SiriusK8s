* <b>Объяснить: в чём разница ClusterIP и NodePort</b> <br>
    ClusterIP и NodePort — это два типа Service в Kubernetes, которые определяют, как приложение становится доступным, и их главное различие заключается в области видимости и способе доступа. ClusterIP является типом по умолчанию: он создает виртуальный IP-адрес, существующий только внутри кластера, поэтому доступ к сервису возможен исключительно изнутри. Этот тип идеально подходит для внутренней коммуникации между микросервисами, например, когда backend должен обращаться к базе данных, которую не следует выставлять наружу. NodePort является надстройкой над ClusterIP: сервис получает внутренний ClusterIP плюс открывает статический порт в диапазоне 30000–32767 на каждой ноде кластера. Это позволяет обращаться к сервису снаружи, используя IP-адрес любой ноды и этот выделенный порт в формате <IP_ноды>:<NodePort>.
