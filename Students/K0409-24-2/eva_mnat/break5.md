В ходе выполнения скрипта 05_systemd_break.sh был создан проблемный systemd-юнит script-server.service. Команда sudo systemctl status script-server показала, что процесс завершается с ошибкой (code=exited, status=1/FAILURE). Просмотр логов через sudo journalctl -u script-server -b --no-pager выявил бесконечный цикл перезапусков. При анализе скрипта /opt/break_lab/script_server.sh обнаружена строка exit 1, заставлявшая сервис завершаться с ошибкой. После исправления значения на exit 0 через sudo nano, выполнения sudo systemctl daemon-reload и перезапуска сервиса (sudo systemctl restart script-server.service) юнит успешно запустился и перестал падать.
