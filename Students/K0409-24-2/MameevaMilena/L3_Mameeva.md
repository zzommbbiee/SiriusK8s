# Блок 1 — Docker networking

Создана изолированная сеть с драйвером bridge. Запущены два контейнера в этой сети: база данных и alpine. Внутри alpine проверена доступность базы данных по имени контейнера через ping и проверку порта. DNS внутри сети разрешил имя контейнера в IP-адрес. Контейнер, запущенный без указания этой сети, не смог найти базу данных по имени.

# Блок 2 — Volumes и persistent data

Создан Docker volume для хранения данных PostgreSQL. Запущен контейнер базы данных с привязкой этого volume, внутри созданы тестовые данные в таблице. Контейнер удален, volume сохранен. При повторном запуске нового контейнера с тем же volume ранее созданные данные успешно восстановлены. Проверено физическое расположение volume на хосте.

# Блок 3 — docker-compose

Создана структура проекта с backend на Flask, frontend на Nginx и PostgreSQL. Написан Dockerfile для backend с установкой зависимостей из requirements.txt. Создан docker-compose файл, связывающий три сервиса с указанием зависимостей, проверок здоровья и общим volume для базы данных. Стек запущен, в базу добавлены тестовые записи. Проверен полный путь запроса от фронтенда через бэкенд к базе данных. Выполнено масштабирование backend до трех экземпляров. Стек остановлен с очисткой volumes.

# Основные выводы

Каждая Docker сеть создает отдельный сетевой namespace со встроенным DNS, позволяя контейнерам в одной сети обращаться друг к другу по имени контейнера, но полностью изолируя их от контейнеров в других сетях. Docker volumes обеспечивают сохранность данных при удалении контейнеров, данные физически хранятся в директории на хосте и могут быть повторно использованы новыми контейнерами. Docker Compose позволяет описывать многоконтейнерное приложение декларативно, автоматически управлять порядком запуска через depends_on с проверками здоровья, масштабировать отдельные сервисы и объединять volumes, сети и конфигурации в одном файле.
