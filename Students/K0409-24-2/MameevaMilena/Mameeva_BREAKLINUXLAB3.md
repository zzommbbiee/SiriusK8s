# 3
В третьей лабораторной работе скрипт искусственно заполнил диск, и требовалось освободить место без перезагрузки системы.


Был выполнен анализ использования дискового пространства с помощью df -h. Для поиска крупных файлов использована команда du -h /var | sort -h | tail, в результате обнаружен файл-заполнитель /var/lib/linux_break_lab/disk_fill.bin, созданный скриптом. Файл был удалён командой sudo rm -f.


После удаления выяснилось, что процесс, удерживающий файл, продолжает занимать место. Из файла /var/lib/linux_break_lab/orphan_holder.pid был получен идентификатор процесса (PID 3182), который затем завершён через sudo kill. Для проверки наличия других удалённых, но удерживаемых файлов использована команда sudo lsof | grep deleted. Также с помощью tune2fs -m 1 был уменьшен зарезервированный процент блоков на разделе для дополнительного освобождения места.


По итогу выполнена повторная проверка df -h — место в корневом разделе успешно освобождено.
