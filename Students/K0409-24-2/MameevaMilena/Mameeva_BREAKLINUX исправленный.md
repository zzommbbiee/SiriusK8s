# 4

В четвёртой лабораторной работе были проанализированы и исправлены неисполняемые файлы в каталоге /opt/break_lab.

С помощью ls -la выведен список файлов и их прав доступа. Для mystery_no_exec отсутствовал флаг x (право на выполнение), поэтому была выполнена команда sudo chmod +x mystery_no_exec, после чего файл успешно запущен.

Файл not_a_binary определён командой file not_a_binary как обычный текстовый ASCII-файл, а не бинарный. Исправить это невозможно, запуск не имеет смысла.

Файл mystery_bad_interpreter при попытке запуска выдал ошибку required file not found. Команда file показала, что это скрипт, использующий интерпретатор /bin/no_such_interpreter_break_lab, которого не существует. С помощью sudo vim mystery_bad_interpreter первая строка была исправлена на #!/bin/bash, после чего скрипт запустился.

Файл mystery_truncated_elf при запуске выдал ошибку формата. Команда file mystery_truncated_elf показала, что это ELF-файл с повреждённой структурой — не удаётся прочитать заголовки разделов. Команда readelf подтвердила, что файл обрезан и не может быть восстановлен.

По итогу все исправимые файлы были починены, повреждённые и некорректные — идентифицированы как не подлежащие запуску.
