## Модуль 2 — Починить сеть

Задача
Восстановить доступ сервера в интернет после контролируемой поломки маршрутизации и DNS. Ответ 200 — выполнение команды curl -I --max-time 10 https://www.avito.ru.

Решение

Сначала проверялся слой маршрутизации:

Выведена таблица маршрутов через ip route, что показало отсутствие default gateway.

Вычислен IP-адрес шлюза исходя из текущей подсети, после чего маршрут был добавлен командой sudo ip route add default via 10.211.55.1.

Для проверки связи с внешним миром был сделан тестовый запрос. В локальной среде Parallels Desktop ICMP-пакеты (ping) наружу блокируются внутренним NAT, поэтому диагностика сразу проводилась по TCP-протоколу через curl.

Выявлена неисправность локального резолвера доменных имен.

Восстановлена цепочка разрешения имен в /etc/nsswitch.conf с помощью потокового редактора sed, вернув параметр dns.

Перезаписан файл /etc/resolv.conf с помощью утилиты tee, где был указан рабочий публичный DNS-сервер 8.8.8.8.

Успешно выполнена целевая проверка доступности тестового домена.

Команды которые открыл для себя

ip route get 8.8.8.8 — показывает, какой именно маршрут и интерфейс система выберет для достижения конкретного адреса. Идеально для траблшутинга (сразу показывает Network is unreachable, если нет дефолтного маршрута).

sed -i 's/^hosts:.*/hosts: files dns/' /etc/nsswitch.conf — потоковая замена текста в файле. Незаменимо в скриптах автоматизации или когда нужно быстро поправить конфиг прямо в консоли.

echo "nameserver 8.8.8.8" | sudo tee /etc/resolv.conf — элегантный способ записать вывод команды в системный файл, требующий root-прав (в связке с sudo tee), так как обычный редирект sudo echo > /etc/file выдаст ошибку прав доступа.

Сложности
Главная проблема в этом задании — файл nsswitch.conf. При проблемах с резолвом доменов сразу лезут править /etc/resolv.conf. Но если в nsswitch.conf в строке hosts: отсутствует параметр dns, система даже не попытается обратиться к DNS-серверам. Без понимания этой иерархии можно очень долго искать причину. Дополнительной сложностью стала особенность сетевого стека гипервизора Parallels Desktop, блокирующая ping.
