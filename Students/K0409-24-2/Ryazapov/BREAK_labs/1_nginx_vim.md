## Модуль 1 — Знакомство с инструментами (Vim/MC) и анализ Nginx логов

Задача
В рамках первого модуля необходимо было продемонстрировать владение консольными редакторами и файловыми менеджерами, а также решить задачу по анализу больших данных. 

Основные цели:
1. Применить нестандартные и продвинутые команды в Vim и MC (включая выполнение внешних команд и навигацию без мыши).
2. Выполнить специфическую задачу в Vim: закомментировать строку, а затем технично ее удалить.
3. Проанализировать сгенерированный access-лог Nginx (около 1 млн строк) и найти топ-10 самых активных IP-адресов.
4. Выполнить аналогичную задачу для лога в формате JSON.

Решение

Для работы с редакторами я сфокусировался на горячих клавишах и встроенных функциях, минимизируя лишние движения:
* В Vim использовал визуальный блочный режим для работы с колонками, а для задачи с удалением применил связку быстрого перехода в начало строки (I), ввода комментария, возврата в нормальный режим (ESC) и удаления строки целиком (dd). Также продемонстрировал чтение вывода команд bash напрямую в буфер редактора.
* В MC активно использовал subshell (скрытую консоль) для просмотра вывода скриптов, чтобы не терять контекст файлового менеджера.

Для анализа логов Nginx:
* Сгенерировал тестовые данные скриптом 01_nginx_log_challenge.sh.
* Для классического лога решил отказаться от стандартного, но медленного пайплайна с двойной сортировкой (awk | sort | uniq -c | sort -nr ). Вместо этого использовал awk с ассоциативными массивами: скрипт прошел по миллиону строк всего один раз, сохранив счетчики в памяти, и только итоговый небольшой массив был отсортирован.
* Для JSON-лога применил утилиту jq для извлечения значений по ключу remote_addr, после чего передал данные в тот же быстрый awk-скрипт.

Команды которые открыл для себя
* Ctrl+v (в Vim) — запускает визуальный блочный режим. Идеально применять для массового комментирования строк или редактирования отступов в коде.
* :r !date (в Vim) — выполняет внешнюю bash-команду (в данном случае date) и вставляет ее вывод прямо в текст файла на следующую строку. Очень удобно для добавления таймстемпов в скрипты.
* Ctrl+o (в MC) — прячет синие панели и открывает полноценный subshell (терминал) с историей команд. Применять, когда нужно посмотреть длинный вывод только что запущенного скрипта.
* awk '{count[$1]++} END {...}' — подсчет уникальных значений через ассоциативный массив. Стоит применять на огромных файлах вместо sort | uniq -c, чтобы сэкономить процессорное время.
* jq -r '.key' — парсинг JSON с выводом "сырого" текста (raw). Флаг -r убирает кавычки вокруг значений, что критически важно для дальнейшей передачи данных в bash-пайплайны.

Сложности

При работе с Vim поначалу было непривычно выполнять "простую" задачу (закомментировать и удалить) без использования мыши — руки по привычке тянулись выделить текст курсором. Однако использование комбинации I -> # -> ESC -> dd оказалось гораздо эффективнее и быстрее на практике.

На этапе парсинга логов главная сложность заключалась в производительности. Первоначальный вариант с sort | uniq -c работал ощутимо долго на файле в 1 миллион строк. Пришлось немного покопать документацию awk, чтобы перенести нагрузку с процессора в оперативную память (использовав массивы), что в итоге ускорило выполнение в несколько раз. Также при работе с JSON логом сначала забыл указать флаг -r в утилите jq — из-за этого IP-адреса выводились в кавычках (например, "192.168.1.1"), что ломало логику группировки и сортировки в последующих командах пайплайна.
