## Модуль 4 — Незапускаемый бинарник

Задача

Провести диагностику и восстановить работоспособность пяти различных исполняемых файлов в директории /opt/break_lab/. Необходимо определить причины отказа в запуске (права доступа, зависимости, ошибки формата) и предложить методы решения.


Решение
Как я делал и мои действия:

1. mystery_no_exec: Обнаружено отсутствие флага исполнения. Исправлено через chmod +x.
2. mystery_bad_interpreter: С помощью file и head выявлен некорректный путь к интерпретатору в shebang. Исправлено заменой строки через sed.
3. mystery_dyn: С помощью ldd и readelf -l проанализированы секции динамического линкера. Выявлено несоответствие путей библиотек (битый INTERP).
4. mystery_truncated_elf: Анализ через readelf -h показал повреждение структуры ELF-заголовка (файл усечен), что делает запуск невозможным.
5. not_a_binary: Выявлена подмена типа файла — текстовый скрипт был замаскирован под бинарный файл.

Команды которые открыл для себя
* file — базовый инструмент для определения реального типа содержимого файла, независимо от его расширения.
* ldd — выводит список общих библиотек, необходимых для работы программы, и их текущее наличие в системе.
* readelf -h — чтение заголовка ELF-файла. Позволяет увидеть архитектуру, точку входа и целостность заголовка.
* strings — извлечение печатаемых символов из бинарного файла. Помогает понять назначение программы, если она повреждена.
* strace — отслеживание системных вызовов в реальном времени. Позволяет увидеть, на каком именно моменте (открытие файла, выделение памяти) падает программа.

Сложности
Самым интересным моментом стало столкновение с ошибкой Exec format error. В отличие от проблем с интерпретатором или правами, здесь проблема заключалась в физической целостности файла. С помощью hexdump удалось подтвердить наличие магических байт ELF, но readelf окончательно подтвердил диагноз — файл был усечен, и его таблица заголовков ссылалась на несуществующие данные вне границ файла. Это научило меня тому, что не всё в Linux можно починить правкой конфигов — иногда файл просто поврежден и требует восстановления из бэкапа или переустановки пакета.
