## Что понял:
----------------
+ Deployment — управляет подами и обеспечивает масштабирование
+ Можно откатывать версии (rollout undo)
+ Service — даёт стабильный доступ к подам
+ Ingress — маршрутизирует HTTP-трафик
+ Есть разные типы Service (ClusterIP, NodePort, LoadBalancer)

**Разница между типами Service:**
+ ClusterIP — доступ только внутри кластера
+ NodePort — открывает порт на каждой ноде
+ LoadBalancer — создаёт внешний IP (в облаке)
