Для диагностики использовала команду file — она показывает тип файла. Если файл ELF то это исполняемый бинарник, если ASCII text то это обычный текст.
Команда ls -l показывает права доступа.
Для файла mystery_no_exec проблема была в отсутствии бита исполнения. Команда chmod +x решила проблему.
Для файла mystery_bad_interpreter команда head -1 показала неправильный shebang (#!/bin/no_such_interpreter_break_lab). Интерпретатора не существует поэтому файл не запускается. Исправила через sed -i '1s|.*|#!/bin/bash|' — заменила первую строку на правильный shebang.
Для третьего файла mystery_truncated_elf команда file показала что ELF усечён (truncated). Размер всего 256 байт вместо полного бинарника. Такой файл невозможно починить.
Команда ldd показывает зависимости бинарника — какие библиотеки ему нужны. Если есть "not found" значит не хватает библиотек.
Команда strace ./файл показывает системные вызовы — видно на каком этапе происходит ошибка.
Для файла not_a_binary команда file показала ASCII text. Это текстовый скрипт без shebang.
