В третьей лабораторной работе нужно было освободить 6GB на корневом диске без перезагрузки сервера.
Сначала проверила сколько места занято командой df -h / 
Для поиска больших файлов использовала: du -sh /var/* | sort -rh | head -10
Команда du показывает размер директорий, sort сортирует по убыванию, head показывает топ-10. Так нашла файл /var/lib/linux_break_lab/disk_fill.bin который занимал 6GB.
Для поиска «удалённых но открытых» файлов использовала: lsof | grep deleted
Эта команда показывает процессы которые держат удалённые файлы — место не освобождается пока процесс не завершён. Нашла процесс break_lab_orphan_holder который держал 512MB.
Для освобождения места выполнила: rm -f /var/lib/linux_break_lab/disk_fill.bin — удалила большой файл; kill $(cat /var/lib/linux_break_lab/orphan_holder.pid) — убила процесс держатель.
Дополнительно уменьшила резерв файловой системы: tune2fs -m 1 /dev/vda4
По умолчанию ext4 резервирует 5% места для root. Команда уменьшила резерв до 1% — это освободило ещё около 1GB.
Проверка через df -h / показала что освободилось больше 6GB.
