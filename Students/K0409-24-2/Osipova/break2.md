Во второй лабораторной работе требовалось восстановить доступ сервера в интернет после преднамеренной поломки сетевых настроек.
Для диагностики использована команда ip route show — показала таблицу маршрутизации. После запуска скрипта поломки маршрут по умолчанию отсутствовал.
Для восстановления маршрута выполнена команда: ip route add default via 192.168.122.1 dev enp1s0
Команда добавила шлюз по умолчанию — через него сервер отправляет пакеты во внешнюю сеть. Без этого маршрута доступ в интернет невозможен.
Файл /etc/resolv.conf отвечает за DNS-настройки. В нём прописаны nameserver'ы (8.8.8.8) которые преобразуют доменные имена в IP-адреса.
Файл /etc/nsswitch.conf управляет порядком разрешения имён. Строка hosts: files dns означает что система сначала проверяет /etc/hosts затем обращается к DNS.
Проверка работоспособности выполнена через curl -I --max-time 10 https://www.avito.ru — сервер ответил HTTP/2 429 что подтверждает успешное восстановление сети.
