Отчет по лабораторной работе: Исследование занятого дискового пространства

Цель работы: Изучить методы обнаружения и очистки дискового пространства, занятого скрытыми (удаленными, но открытыми) файлами.

Ход работы:

    Создание проблемы (заполнение диска):
    Был запущен скрипт 03_disk_break.sh, который создал файл-заполнитель /var/lib/linux_break_lab/disk_fill.bin размером 6144 MiB (6.1 ГБ). Скрипт также запустил фоновый процесс break_lab_orphan_holder (PID 63592), который удерживает файл открытым.

    Проверка состояния диска:
    Команда df -h / показала, что файловая система /dev/nvme0n1p4 имеет размер 34 ГБ, из которых использовано 18 ГБ (55%).

    Анализ содержимого директории:
    С помощью команды sudo du -sh /var/lib/linux_break_lab/* было подтверждено наличие файла disk_fill.bin объемом 6.1 ГБ.

    Попытка удаления файла:
    Была выполнена команда sudo rm -f /var/lib/linux_break_lab/disk_fill.bin для удаления файла-заполнителя.

    Поиск «скрытого» использования диска:
    После удаления файла место на диске может не освободиться, если процесс продолжает его использовать. Для поиска таких файлов была использована команда sudo lsof | grep deleted.
    В выводе было обнаружено множество процессов (NetworkManager, python3, gdm, pipewire, firefox и др.), удерживающих удаленные файлы в памяти (статус deleted).

Вывод:
В ходе работы продемонстрировано, что простое удаление файла через rm не всегда приводит к немедленному освобождению места, если файл открыт каким-либо процессом. Для полного освобождения пространства необходимо либо перезапустить соответствующую службу, либо завершить процесс, удерживающий дескриптор файла.
