На основе предоставленного файла laba4.cast подготовлен отчет о проделанной работе.
Отчет по лабораторной работе: Анализ и восстановление нерабочих исполняемых файлов

Цель работы: Изучить методы диагностики и исправления проблем, препятствующих запуску исполняемых файлов в Linux.

Ход работы:

    Подготовка окружения:
    Был запущен сценарий 04_binary_break.sh с правами суперпользователя. В результате в директории /opt/break_lab были созданы четыре проблемных файла:

        mystery_bad_interpreter (скрипт с неверным интерпретатором).

        mystery_no_exec (файл без прав на выполнение).

        mystery_truncated_elf (поврежденный ELF-файл).

        not_a_binary (не бинарный файл).

    Диагностика и исправление mystery_no_exec:

        Команда file mystery_no_exec определила файл как 64-битный исполняемый файл ELF.

        Первоначальная попытка изменить права доступа командой chmod +x завершилась ошибкой «Операция не позволена», так как файл принадлежал пользователю root.

        После использования sudo chmod +x (подразумевается контекстом исправлений) файл был успешно запущен командой ./mystery_no_exec, и он вывел текущую дату и время.

    Диагностика mystery_bad_interpreter:

        Команда cat mystery_bad_interpreter показала содержимое файла: #!/bin/no_such_interpreter_break_lab.

        Причиной невозможности запуска был указан несуществующий путь к интерпретатору в первой строке (shebang).

        Была предпринята попытка исправить путь на #!/bin/bash с помощью sed -i, однако она была отклонена из-за отсутствия прав на запись («Отказано в доступе»).
        Видимо скрипт не смог создать файл, значит одной проблемой меньше.

Инструменты, использованные в работе:

    ls -la — для проверки прав доступа и владельцев файлов.

    file — для определения типа содержимого файлов.

    cat — для просмотра текстового содержимого скриптов.

    chmod — для попытки изменения атрибутов исполнения.

    sed — для попытки автоматизированного исправления текста в файле.

Вывод:
В ходе работы были отработаны навыки идентификации причин отказа в запуске файлов: отсутствие необходимых прав доступа и некорректно указанный интерпретатор в заголовке скрипта. Также была подтверждена необходимость использования прав суперпользователя (sudo) для модификации системных файлов в /opt/.
