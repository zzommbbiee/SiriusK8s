Описание хода выполнения лабораторной работы №2
Общие сведения

В данной работе производилась настройка и проверка работоспособности веб-сервера Nginx в среде Linux. Основной упор был сделан на проверку конфигурационных файлов, управление системными службами и верификацию ответов сервера через терминал.
Этапы выполнения работы
1. Тестирование конфигурации сервера

Перед применением изменений была выполнена проверка синтаксиса основного конфигурационного файла Nginx.

    Использованная команда: sudo nginx -t.

    Результат: Система подтвердила корректность синтаксиса и успешное прохождение теста конфигурации.

2. Управление состоянием службы

Для вступления в силу обновленных параметров конфигурации был произведен перезапуск процесса веб-сервера.

    Использованная команда: sudo systemctl restart nginx.

    Действие: Служба nginx была успешно перезагружена через системный менеджер служб systemctl.

3. Верификация доступности веб-ресурса

Для подтверждения того, что сервер корректно обрабатывает входящие запросы, была выполнена проверка связи с локальным узлом.

    Использованная команда: curl -I http://127.0.0.1.

    Метод: Запрос только заголовков (флаг -I) для минимизации вывода и быстрой проверки статус-кода.

    Результат: Получен ответ HTTP/1.1 200 OK, свидетельствующий о полной работоспособности веб-сервера.

Заключение

По итогам сессии была подтверждена стабильная работа веб-сервера с текущими настройками. Все команды выполнены без ошибок, доступ к локальному ресурсу обеспечен в полном объеме.
