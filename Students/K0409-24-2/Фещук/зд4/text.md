# БЛОК 1

В kube-system всегда должны быть в статусе Running:

coredns — DNS для кластера
etcd — хранилище данных кластера
kube-apiserver — API сервер
kube-controller-manager — менеджер контроллеров
kube-scheduler — планировщик подов
kube-proxy — сетевые правила на каждом узле
storage-provisioner — провайдер томов (в minikube)
Если хотя бы один из них не Running — кластер неработоспособен.
