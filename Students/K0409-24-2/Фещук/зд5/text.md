ClusterIP — это внутренний IP внутри кластера.
Доступ к сервису есть только у других подов внутри Kubernetes,
снаружи кластера его не видно. Используется для внутреннего взаимодействия
между компонентами.

NodePort — открывает порт на каждом узле (ноде) кластера
в диапазоне 30000-32767. Через этот порт можно обратиться к сервису
снаружи кластера, зайдя по адресу любой ноды: http://<ip_ноды>:<порт>.
По сути, это надстройка над ClusterIP: NodePort создаёт ClusterIP
автоматически и пробрасывает на него трафик с внешнего порта.
