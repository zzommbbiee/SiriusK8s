# Лабораторная работа 2 — Починить сеть

## Задача

Сервер не имеет доступа в интернет, его IP‑адрес известен. Необходимо настроить маршрутизацию, шлюз по умолчанию и DNS‑резолвинг через файлы `resolv.conf` и `nsswitch.conf`, а затем проверить доступ к внешнему сайту с помощью `curl www.avito.ru`.

---

## Подготовка и диагностика сети

Сначала я проверила текущие сетевые настройки сервера. Для этого выполнила:

```bash
ip a
ip route
```

Команда `ip a` показала список сетевых интерфейсов, их состояние и назначенные IP‑адреса. По выводу `ip route` я увидела, что маршрут по умолчанию отсутствует (или указан неправильно), поэтому трафик в интернет не мог уходить через нужный шлюз.

Я также проверила, есть ли вообще связь с внешними адресами:

```bash
ping -c 4 8.8.8.8
ping -c 4 www.avito.ru
```

Пинги не проходили, а `ping www.avito.ru` ещё и не мог разрешить доменное имя — это подтвердило проблемы и с маршрутом, и с DNS.

---

## Настройка маршрутизации и шлюза

Далее я настроила маршрут по умолчанию, указав корректный шлюз. Для этого использовала команду вида:

```bash
sudo ip route add default via IP_шлюза dev интерфейс
```

где:
- `<IP_шлюза>` — IP‑адрес маршрутизатора/шлюза в локальной сети;
- `<интерфейс>` — сетевой интерфейс сервера (например, `ens33`, `eth0` и т.п.).

После добавления маршрута я ещё раз проверила таблицу маршрутизации:

```bash
ip route
```

В выводе появилась строка `default via IP_шлюза dev интерфейс`, что означало корректно настроенный маршрут по умолчанию.

---

## Настройка DNS (resolv.conf и nsswitch.conf)

Далее я настроила DNS‑резолвинг. Для этого отредактировала файл `/etc/resolv.conf`:

```bash
sudo vim /etc/resolv.conf
```

И прописала в нём DNS‑серверы, например:

```conf
nameserver 8.8.8.8
nameserver 8.8.4.4
```

Затем я проверила файл `/etc/nsswitch.conf`, чтобы убедиться, что система обращается к DNS‑серверу при поиске хостов:

```bash
sudo vim /etc/nsswitch.conf
```

В нём нашла строку:

```conf
hosts: files dns
```

Такой порядок означает, что сначала проверяются локальные файлы (`/etc/hosts`), а затем выполняется запрос к DNS‑серверу. Если там было что‑то другое, я приводила строку к виду `hosts: files dns`.

---

## Проверка работоспособности сети

После настройки маршрута и DNS я снова проверила связь. Сначала — с внешним IP‑адресом:

```bash
ping -c 4 8.8.8.8
```

Теперь ответы приходили, что означало исправно работающий маршрут до интернет‑шлюза.

Затем я проверила разрешение доменного имени и доступ к сайту:

```bash
ping 8.8.8.8
```

Команда `ping` успешно резолвила домен в IP‑адрес и получала ответы, а `curl` возвращал HTML‑код главной страницы сайта без ошибок. Это подтвердило, что и маршрутизация, и DNS настроены корректно.

Основная сложность заключалась в том, чтобы последовательно проверить все возможные причины отсутствия интернета: интерфейс, маршрут и DNS.
