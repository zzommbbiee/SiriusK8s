Модуль 3 — Очистка дискового пространства
Задача
Восстановить работоспособность системы путем освобождения места на корневом разделе. Основная сложность заключалась в том, что часть места удерживалась процессом даже после удаления файла, и это нужно было исправить без перезагрузки.

Решение
Для выполнения задачи были предприняты следующие действия:

С помощью команды df -h зафиксировано полное отсутствие свободного места (100% занято) на разделе /.

С помощью du -sh проанализированы директории и найден основной источник нагрузки в /var/lib/linux_break_lab.

Удален основной файл-заполнитель disk_fill.bin объемом 5.5 ГБ.

После удаления обнаружено, что место освободилось не полностью. Использована команда lsof | grep deleted для поиска процессов, удерживающих удаленные файлы.

Выявлен процесс sleep (PID 6815), удерживающий файл orphan_blob (512 МБ).

Процесс был завершен командой kill 6815, после чего всё зарезервированное место (итого 6.0 ГБ) вернулось в систему.

Запись сессии: https://asciinema.org/a/YyTdn71Y4WbJLIKN
Команды которые открыл для себя
df -h — быстрая проверка свободного места на всех смонтированных разделах.

du -sh [путь] — оценка реального размера папки или файла. Удобно для поиска «тяжелых» мест.

lsof | grep deleted — незаменимая вещь для поиска файлов, которые удалены с диска, но всё еще занимают место, так как открыты каким-то процессом.

kill [PID] — принудительное завершение процесса, чтобы освободить удерживаемые им ресурсы.

Сложности

Основной момент — не забыть проверить систему через lsof после удаления файлов, так как цифры в df могут не сойтись из-за открытых дескрипторов. Когда «убил» нужный процесс, место сразу пришло в норму.
