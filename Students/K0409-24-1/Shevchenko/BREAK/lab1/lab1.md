2. ___BREAK___\
Так ну тут я сначала в файл, созданный через редактор nano вставил скрипт с методички, там некоторые строки убрать пришлось, связанные со скриптом common.sh. Тогда я еще не знал что скрипт common.sh оказывается лежит на гите. 
После создания скрипта я файл делаю исполняемым через chmod +x. Потом запускаю как скрипт через ./ 
Ну и в результате создаются два файла в /var/log/nginx - access.log и access_json.log. 
Чтобы найти 10 адресов самых активных юзается команда awk '{print $1}' /var/log/nginx/access.log | sort | uniq -c | sort -nr | head -n 10. Это для обычного лога. Тут насколько я понял uniq -c удаляется дубликаты и считает сколько их, а потом sort -nr делает сортировку по убыванию по количеству вот этих дубликатов
Для json юзается jq -r '.remote_addr' /var/log/nginx/access_json.log | sort | uniq -c | sort -nr | head -n 10. Тут тоже самое с сортировкой. Флаг -r тут типо убирает оформление команды jq, она по умолчанию в кавычках пишет.
Ну а в конце просто удаляем оба файла логов чтобы место не занимали через sudo rm -f /var/log/nginx/access.log /var/log/nginx/access_json.log.
Вот ссылка на видос: https://asciinema.org/a/RGNVcRCRsOaqVXAK
