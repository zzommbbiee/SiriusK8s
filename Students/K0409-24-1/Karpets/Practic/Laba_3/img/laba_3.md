На скриншоте видна команда `docker compose ps`. Она показывает список работающих контейнеров. В данном случае работают два контейнера — приложение и база данных. У обоих статус «healthy», то есть всё работает нормально. Команда нужна для быстрой проверки, запустились ли контейнеры без ошибок.

![alt text](image_2026-04-01_13-57-48.png)
На скриншоте видна команда `curl --noproxy "**" localhost:8080/api/items`. Она отправляет запрос к приложению, которое работает на этом же компьютере через порт 8080, и просит показать список товаров по адресу `/api/items`. В ответ приходит список: `apple`, `banana`, `cherry` с их номерами. Команда нужна, чтобы проверить, что приложение правильно отвечает на запросы и возвращает данные.

![alt text](image_2026-04-01_13-58-02.png)
Первая команда: `docker compose up -d --scale backend=3`  
Она запускает контейнеры в фоновом режиме и увеличивает количество копий приложения (backend) с одной до трёх. То есть теперь работает три одинаковых контейнера с приложением. База данных и фронтенд (веб-интерфейс) запускаются по одному разу.
Вторая команда: `docker compose ps`  
Она показывает список всех работающих контейнеров.  
В выводе видно:
- Три контейнера `backend` (номера 1, 2, 3) — все работают
- Один контейнер с базой данных `db`
- Один контейнер с фронтендом `frontend`, который доступен с компьютера через порт 8080
Команда с `--scale backend=3` нужна, чтобы запустить несколько копий одного и того же приложения. Это делается для нагрузки — если приложение получает много запросов, три копии могут обработать их быстрее, чем одна. Команда `ps` нужна, чтобы проверить, что все три копии успешно запустились и работают.
