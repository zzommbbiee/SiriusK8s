Первым делом я проверила текущее состояние сети. Командой ip addr show я посмотрела все сетевые интерфейсы и их IP-адреса. 
Затем командой ip route show я проверила таблицу маршрутизации. Эта таблица говорит компьютеру, куда отправлять сетевые пакты в зависимости от их назначения.

Я попробовала выполнить ping 8.8.8.8. В ответ я получила ошибку "Сеть недоступна" — это подтвердило, что маршрута в интернет действительно нет.

Также я проверила файл /etc/resolv.conf командой cat. Этот файл хранит адреса DNS-серверов — специальных серверов, которые преобразуют имена сайтов (например, 
www.avito.ru) в IP-адреса. В файле был указан нерабочий DNS-сервер.
Мне нужно было добавить маршрут по умолчанию — указать системе, через какой шлюз выходить в интернет.

Сначала я попробовала шлюз 192.168.190.1. Это самый распространённый адрес шлюза — часто роутеры используют первый адрес в подсети. Команда для добавления 
маршрута: sudo ip route add default via 192.168.190.1 dev ens160. Ключевые слова здесь: "default" означает маршрут по умолчанию, "via" указывает адрес шлюза, 
"dev" — сетевой интерфейс. 
Я проверила доступность шлюза командой ping -c 3 192.168.190.1. Пакеты уходили но возвращались. Но когда 
я попробовала пропинговать 8.8.8.8, пакеты не проходили. Я выполнила nslookup www.avito.ru — это команда для проверки DNS-разрешения, она тоже не сработала, 
выдав ошибку. Значит, шлюз 192.168.190.1 доступен, но не пропускает трафик дальше в интернет. Такое бывает, если на шлюзе не настроен NAT.

Тогда я удалила этот маршрут командой sudo ip route del default и попробовала другой шлюз — 192.168.190.2. Я добавила его той же командой: sudo ip route add 
default via 192.168.190.2 dev ens160. После этого снова выполнила ping -c 3 8.8.8.8. Пинг прошел.
Теперь, когда маршрутизация заработала, нужно было настроить DNS. Я исправила /etc/resolv.conf командой:

sudo bash -c 'cat > /etc/resolv.conf << EOF
nameserver 8.8.8.8
nameserver 1.1.1.1
EOF'

Разберу эту команду: sudo bash -c запускает новую оболочку с правами администратора, cat > /etc/resolv.conf записывает то, что идёт дальше, в файл, а << EOF — 
позволяет ввести многострочный текст. В результате в файл записались две строки: "nameserver 8.8.8.8" (DNS-сервер Google) и "nameserver 1.1.1.1".

После всех настроек я выполнила финальную проверку. Командой nslookup www.avito.ru я убедилась, что DNS работает — имя сайта успешно преобразовалось в IP-адрес. 
Командой curl www.avito.ru я загрузила веб-страницу. Так же прошел и пинг.
